# 🕺 摇摆舞动作分析器 - 产品与技术方案

## 一、产品设计方案

### 1.1 产品定位
**一句话描述**: 帮助摇摆舞者通过 AI 对比自己与大师的动作差异，获得针对性改进建议的移动应用。

**目标用户**:
- 摇摆舞学习者（初级-中级）
- 想提升技术的业余爱好者
- 需要教学辅助的舞蹈老师

### 1.2 版本规划

#### MVP (版本 1.0) - 图片对比
**核心功能**:
1. 上传两张跳舞照片（自己 vs 大师）
2. AI 自动识别画面中的人物（支持 1-2 人）
3. 用户选择要对比的舞者
4. 姿态关键点提取与可视化叠加
5. 差异分析报告（角度、姿态、重心）
6. 改进建议生成

**用户流程**:
```
打开App → 上传我的照片 → 上传大师照片
→ AI识别人物 → 选择对比对象（左/右）
→ 查看骨架叠加对比 → 阅读分析报告
→ 保存/分享结果
```

#### 版本 2.0 - 视频对比
**新增功能**:
1. 上传两段跳舞视频
2. 关键帧自动提取（基于动作变化）
3. 多人物追踪（支持双人舞选择）
4. 时序动作对比（动作流畅度、节奏）
5. 动态轨迹可视化（运动路径）
6. 历史记录管理

### 1.3 界面设计

**核心页面**:
- 首页：上传入口（拍照/相册/视频）
- 对比页：双图/双视频展示 + 控制面板
- 分析页：骨架叠加 + 数据面板 + 建议
- 历史页：过往分析记录

**关键交互**:
- 拖动调整透明度（叠加视图）
- 点击切换关键点显示
- 缩放/平移查看细节
- 逐帧控制（视频版）

### 2.2 技术栈选择

| 层级 | 技术 | 选择理由 |
|------|------|----------|
| **前端** | React + TypeScript + Tailwind CSS | 开发效率高，PWA 支持好 |
| **移动端** | PWA (Progressive Web App) | 无需应用商店审核，跨平台 |
| **后端** | Python FastAPI | AI 生态丰富，异步性能优秀 |
| **AI 推理** | MediaPipe Pose | 轻量、实时、支持移动端 |
| **图像处理** | OpenCV + Pillow | 标准库，功能完善 |
| **数据库** | SQLite (MVP) → PostgreSQL | 初期轻量，后期扩展 |
| **部署** | Vercel (前端) + Render (后端) | 免费额度足够 MVP |

### 2.3 AI 模型方案

#### 姿态估计 - MediaPipe Pose
- **优势**: Google 出品，轻量级，浏览器可运行，33个关键点
- **精度**: 适合舞蹈动作分析
- **性能**: 移动端 30fps
- **成本**: 免费

#### 多人检测 - YOLOv8
- **用途**: 识别画面中的多个人物
- **输出**: 边界框 + 人物 ID
- **成本**: 自托管免费

#### 分析建议 - GPT-4 / Claude API
- **输入**: 姿态差异数据（角度、距离）
- **输出**: 自然语言改进建议
- **成本**: 每次分析约 $0.01-0.05

### 2.4 数据处理流程

```
图片上传
  ↓
人物检测 (YOLO) → 返回人物数量和位置
  ↓
用户选择对比的人物
  ↓
姿态估计 (MediaPipe) → 提取33个关键点
  ↓
关键点对齐（标准化坐标系）
  ↓
差异计算（角度、距离、重心）
  ↓
可视化叠加渲染
  ↓
AI 生成改进建议
  ↓
返回完整分析报告
```

## 三、成本评估

### 3.1 开发成本

#### 人力投入

| 阶段 | 工作量 | 说明 |
|------|--------|------|
| **产品设计** | 3 天 | 交互设计、视觉设计、原型 |
| **前端开发** | 10 天 | PWA 页面、Canvas 渲染、上传组件 |
| **后端开发** | 7 天 | API、数据库、文件存储 |
| **AI 集成** | 5 天 | MediaPipe 集成、差异算法、建议生成 |
| **测试优化** | 3 天 | 跨设备测试、性能优化 |
| **部署上线** | 2 天 | CI/CD、域名配置、SSL |
| **总计** | **~30 天** | 全职单人开发 |

## 四、风险评估

### 技术风险
| 风险 | 可能性 | 应对方案 |
|------|--------|----------|
| MediaPipe 精度不足 | 中 | 准备 OpenPose 备选方案 |
| 移动端性能问题 | 中 | 云端处理 + 结果回传 |
| 多人检测混乱 | 中 | 让用户手动框选人物 |

### 商业风险
| 风险 | 可能性 | 应对方案 |
|------|--------|----------|
| 舞蹈类型扩展困难 | 中 | MVP 专注摇摆舞，后期扩展 |
| AI 成本过高 | 低 | 提供付费订阅，限制免费次数 |

## 六、建议

### 是否值得开发？
✅ **推荐开发**，原因：
1. 技术可行性强（MediaPipe 成熟）
2. 开发成本可控（1 个月单人）
3. 运维成本低（初期 < ¥1000/月）
4. 有明确的用户群体

### 开发策略
1. **先做 MVP**（图片对比），验证需求
2. **收集反馈**后再做视频版本
3. **考虑 Freemium 模式**：免费 3 次/天，付费无限次
